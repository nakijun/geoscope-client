Пользователь.

Пользователь это компонент системы, через который лицо или служба получает доступ к данным. Клиент регистрируясь, создает уникального пользователя со своим именем (логином) и паролем. При этом пользователю назначаются два собственных файла-безопасности (публичный и приватный), ключи доступа, а также некоторый объем ресурсов системы, в пределах которых возможно создание объектов.

Основные пункты:

"имя (логин)" - пункт, показывающий уникальное имя пользователя в системе. Изменение данного поля невозможно, дается при регистрации.

"ОПИСАНИЕ","КонтактИнфо","ПАРОЛЬ" - пункты, устанавливающие контактную информацию и пароль. Доступны для изменения самим пользователем в любое время.

"стартовый компонент пользователя" - пункт, устанавливающий компонент, панель которого будет открываться сразу же после входа в систему. Изменить его можно через контекстное меню кнопки.

"файлы безопасности" - пункты, устанавливающие, какой файл безопасности будет использоваться при создании нового объекта пользователя, и какой будет как приватный.

"ресурсы пользователя" - группа пунктов, описывающих сколько и каких ресурсов выделено пользователю и сколько им уже израсходовано. Существуют несколько типов ресурсов: максимальный объем памяти сервера доступного для пользователя, максимальная суммарная площадь создаваемых им объектов на карте, и максимальная площадь единичного объекта, который он может создать. Пункты недоступны для изменения пользователем, но доступны администратору системы.

"список почтовых ящиков" - кнопка, открывающая список всех почтовых ящиков пользователя (внутренняя почта проекта).

"Чат" - кнопка, открывающая окно, позволяющее обмениваться мгновенными сообщениями с данным пользователем, если он в системе.

"Папка" - кнопка, открывающая окно просмотра пользовательской папки, которая является папкой файловой системы на стороне сервера, закрепленной за данным пользователем. Доступ к ней пользователь имеет также через веб-браузер.

"ключи доступа" - список ключей безопасности принадлежащих пользователю. Имея ключи, он имеет доступ только к тем объектам, в файле безопасности которых, содержатся такие же ключи. Данный список формируется администратором.

"Редактор кодовых ключей" - пункт, редактирующий так называемые кодовые ключи доступа. Наряду с постоянными ключами, пользователь может иметь и "кодовые", которые создаются/добавляются/удаляются не администратором, а самим пользователем через генерируемый уникальный код (или пароль).
